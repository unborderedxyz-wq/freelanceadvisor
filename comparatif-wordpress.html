Les idées reçues qui coûtent cher

"WordPress c'est gratuit, donc pas besoin de budget." WordPress est gratuit. Un site WordPress professionnel ne l'est pas. Entre l'hébergement, le thème, les plugins premium, et surtout le temps de développement — un site WP sérieux coûte entre 3 000€ et 15 000€. Le "gratuit" c'est pour les blogs perso, pas pour les entreprises.

"Tous les développeurs WordPress se valent." Il y a un monde entre quelqu'un qui installe Elementor et glisse des blocs, et un développeur qui code des thèmes custom en PHP, crée des plugins sur mesure, et optimise les requêtes de base de données. Le premier facture 200€/jour, le second 500€ — et le résultat n'a strictement rien à voir.

"Un page builder ça suffit." Elementor, Divi, WPBakery — ils ont leur place pour du prototypage rapide. Mais pour un site professionnel qui doit performer, ces outils ajoutent des tonnes de code inutile, ralentissent le site, et créent une dette technique monstrueuse. Un bon développeur code proprement sans page builder.

"30 plugins c'est normal." Non. Chaque plugin est un vecteur de faille de sécurité, un ralentisseur de performance, et une dépendance supplémentaire. Un bon développeur WordPress fait avec 8-10 plugins max et code le reste en natif.

"WordPress c'est pas sécurisé." WordPress n'est pas insécurisé. Les sites WordPress mal configurés le sont. Mises à jour négligées, plugins abandonnés, mots de passe faibles, hébergement bas de gamme — c'est ça qui crée les failles, pas WordPress lui-même.

Ce que la plupart des clients ne savent pas

1. Un développeur qui code sans page builder livre un site 3x plus rapide. Les page builders génèrent du HTML/CSS surchargé. Un thème custom codé à la main pèse 10x moins lourd. Sur mobile, la différence est flagrante — et Google le sait (Core Web Vitals).

2. La sécurité WordPress se joue dès l'installation. Configuration du fichier .htaccess, headers HTTP, désactivation de XML-RPC, changement du préfixe de base de données, authentification deux facteurs — ce sont les basiques qu'un vrai développeur met en place dès le jour 1. Pas après le premier hack.

3. WooCommerce peut tout faire, mais pas tout seul. WooCommerce est puissant, mais il nécessite un développeur qui comprend ses hooks, ses templates, et ses limites. Un WooCommerce bien configuré par un expert rivalise avec Shopify en performance. Un WooCommerce mal configuré est un cauchemar.

Les critères qui comptent vraiment

Maîtrise technique WordPress (35%). Thèmes custom PHP, création de plugins, hooks/filters, Custom Post Types, ACF avancé, REST API. Le test : "Montrez-moi un plugin que vous avez développé." Si la réponse est "j'utilise Elementor Pro", passez votre chemin.

Compétences complémentaires (25%). SEO technique, sécurité WordPress, performance, UX/conversion. Un site WordPress beau mais lent et invisible sur Google, c'est un échec.

Communication et réactivité (25%). Temps de réponse, qualité du reporting, pédagogie technique.

Garanties et suivi (15%). Support post-livraison, maintenance incluse, correction de bugs.

Les erreurs à éviter

  • Choisir un "webmaster" qui ne sait pas coder — installer un thème et 30 plugins n'est pas du développement
  • Ignorer les performances — un site WordPress à 4 secondes de chargement perd la moitié de ses visiteurs
  • Négliger la sécurité — WordPress est la cible n°1 des hackers, la sécurisation doit être dans le cahier des charges
  • Ne pas demander le code source — si votre freelance refuse de vous donner accès au code, fuyez
  • Confondre page builder et développement — demandez si le freelance code en PHP ou s'il drag & drop

Questions fréquentes

Combien coûte un freelance WordPress ?

Entre 250€ et 650€/jour pour un freelance, 800 à 1500€/jour en agence. Un senior qui code des thèmes et plugins custom se situe autour de 400-600€. En dessous de 300€, vous aurez un installateur de thèmes, pas un développeur.

WordPress ou Shopify pour un e-commerce ?

WordPress + WooCommerce si vous voulez un contrôle total, des fonctionnalités très spécifiques, et que vous avez un développeur. Shopify si vous voulez simplicité et rapidité sans gérer l'hébergement. Pour les deux, le prestataire compte plus que la plateforme.

Comment vérifier les compétences ?

Demandez : 1) un thème ou plugin custom qu'il a codé en PHP, 2) 2-3 anciens clients joignables, 3) un score PageSpeed d'un de ses sites. Bonus : demandez combien de plugins il utilise en moyenne sur ses projets — moins c'est mieux.

Combien de temps prend un projet WordPress ?

Site vitrine custom : 2 à 4 semaines. E-commerce WooCommerce : 4 à 8 semaines. Refonte complète avec migration de contenu : 3 à 6 semaines. Ajoutez toujours 20% de marge pour les imprévus.
